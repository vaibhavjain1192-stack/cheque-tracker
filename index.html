Setup guide
Five steps to get fully free, automated cheque tracking running.
1

Get a free Gemini API key

Go to aistudio.google.com and sign in with your Google account.

  • Click Get API key → Create API key in new project
  • Copy the key (starts with AIza)
  • Paste it in the API configuration box on the Upload tab

✓ Free: 15 requests/minute · 1,500 requests/day · No credit card needed

2

Create your Google Sheet

Go to sheets.google.com → create a blank spreadsheet.

In Row 1, add exactly these headers (one per column):

Vendor | Amount | Cheque Date | Reminder Date | Account No | Bank | MICR Code | Status | Uploaded At

Tip: Copy the sheet URL from your browser — you'll need it in Step 4.

3

Open Apps Script editor

In your Google Sheet: click Extensions → Apps Script.

  • Delete the default function myFunction() {} code
  • Go to the Apps Script Code tab (sidebar) → copy the full script
  • Paste it into the Apps Script editor
  • Click Save (💾 icon or Ctrl+S)
4

Deploy as a Web App

In Apps Script: click Deploy → New deployment → select type Web app.

  • Description: Cheque Tracker API
  • Execute as: Me
  • Who has access: Anyone
  • Click Deploy → Authorize access (grant Gmail + Sheets permissions)
  • Copy the Web app URL (looks like https://script.google.com/macros/s/AKfy.../exec)
  • Paste it into the Google Sheets URL field on the Upload tab
5

Set up the daily reminder trigger

In Apps Script: click the clock icon (Triggers) in the left sidebar → + Add Trigger.

  • Choose function: checkReminders
  • Event source: Time-driven
  • Type: Day timer
  • Time: 9am to 10am
  • Click Save

✓ Done! Every morning at 9 AM, the script checks all cheques and emails you 2 days before each one is due.

Google Apps Script code
Copy this entire script into your Apps Script editor (Extensions → Apps Script).
📋 Full Apps Script (paste into editor)
// ═══════════════════════════════════════════════ // Cheque Tracker — Google Apps Script // Paste this into Extensions → Apps Script // ═══════════════════════════════════════════════ // ── CONFIG: edit these two lines ── const NOTIFY_EMAIL = 'your@gmail.com'; const DAYS_BEFORE = 2; const SHEET_NAME = 'Sheet1'; // ── Helper: CORS headers for browser requests ── function corsResponse(data) { return ContentService .createTextOutput(JSON.stringify(data)) .setMimeType(ContentService.MimeType.JSON); } // ── doGet: handles GET requests from browser (CORS-friendly) ── function doGet(e) { try { if (e.parameter && e.parameter.data) { // Data sent as URL parameter const data = JSON.parse(decodeURIComponent(e.parameter.data)); appendToSheet(data); return corsResponse({ ok: true, method: 'GET' }); } // Health check when no data return corsResponse({ status: 'Cheque Tracker running', ok: true }); } catch(err) { return corsResponse({ ok: false, error: err.message }); } } // ── doPost: handles POST requests ── function doPost(e) { try { const data = JSON.parse(e.postData.contents); appendToSheet(data); return corsResponse({ ok: true, method: 'POST' }); } catch(err) { return corsResponse({ ok: false, error: err.message }); } } // ── appendToSheet: shared logic for doGet and doPost ── function appendToSheet(data) { const sheet = SpreadsheetApp .getActiveSpreadsheet() .getSheetByName(SHEET_NAME); // Auto-create headers if sheet is empty if (sheet.getLastRow() === 0) { sheet.appendRow([ 'Vendor', 'Amount', 'Cheque Date', 'Reminder Date', 'Account No', 'Bank', 'MICR Code', 'Status', 'Saved At' ]); } sheet.appendRow([ data.vendor || '', data.amount || '', data.chequeDate || '', data.reminderDate || '', data.accountNo || '', data.bankName || '', data.micrCode || '', 'Pending', new Date().toLocaleString() ]); } // ── checkReminders: run daily via trigger ── function checkReminders() { const sheet = SpreadsheetApp .getActiveSpreadsheet() .getSheetByName(SHEET_NAME); const rows = sheet.getDataRange().getValues(); const today = new Date(); today.setHours(0, 0, 0, 0); rows.slice(1).forEach((row, i) => { if (row[7] === 'Notified') return; const remDate = new Date(row[3]); remDate.setHours(0, 0, 0, 0); const diff = (remDate - today) / 86400000; if (diff >= 0 && diff < 1) { GmailApp.sendEmail( NOTIFY_EMAIL, `⚠️ Cheque due in ${DAYS_BEFORE} days: ${row[0]}`, `Vendor: ${row[0]}\nAmount: ₹${row[1]}\nDue: ${row[2]}\nMICR: ${row[6]||'N/A'}\nBank: ${row[5]||'N/A'}` ); sheet.getRange(i + 2, 8).setValue('Notified'); } }); } // ── sendTestEmail: run manually once to verify Gmail works ── function sendTestEmail() { GmailApp.sendEmail(NOTIFY_EMAIL, '✅ Cheque Tracker working!', 'Setup is correct.'); Logger.log('Test email sent to ' + NOTIFY_EMAIL); }

🔧 Step-by-step setup
1

Create a free GitHub account

Go to github.com → Sign up. Use your Gmail. It's free.

2

Create a new repository

Click the + icon (top right) → New repository.

  • Repository name: cheque-tracker
  • Visibility: Public (required for free Pages)
  • Check Add a README file
  • Click Create repository
3

Upload your HTML file

In your new repo, click Add file → Upload files.

  • Drag and drop cheque-tracker-free.html
  • Important: rename it to exactly index.html before uploading — this makes it your homepage
  • Scroll down → click Commit changes
4

Enable GitHub Pages

In your repo, click Settings (top tab) → scroll to Pages (left sidebar).

  • Source: Deploy from a branch
  • Branch: main → folder: / (root)
  • Click Save

Wait ~60 seconds, then your site is live at:

https://yourusername.github.io/cheque-tracker
5

Update your Google Apps Script CORS setting

Since you're now on HTTPS, go to your Apps Script → Deploy → Manage deployments → Edit.

  • No changes needed to the code — just create a new version
  • Click Deploy to refresh the Web App URL
  • Paste the new URL into your hosted app's config

✓ That's it! Your cheque tracker is now live on the internet.

6

How to update the app later

Whenever you get an updated index.html from Claude:

  • Go to your GitHub repo → click index.html
  • Click the pencil icon (Edit) or upload a new file
  • Commit the change → GitHub auto-deploys in ~30 seconds
🔒 Security note: API key on GitHub
⚠ Never hardcode your API key in the HTML file.
The app already handles this safely — the API key is entered by the user at runtime and stored only in their browser's localStorage. The HTML file uploaded to GitHub contains no secrets. Safe to make the repo public.
